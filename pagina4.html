Critérios de Aceitação 1. O processo de login deve exigir apenas um email e uma senha. 2. O sistema deve oferecer a opção de “lembrar-me” para evitar a necessidade de login frequente. 3. Em caso de senha esquecida, deve haver um link fácil para redefinição de senha, que envie um email com instruções de redefinição. 4. Após o login bem sucedido, o usuário deve ser redirecionado à página inicial. Fora do Escopo 1. Integração com sistemas de login de terceiros (como o google, linkedIn). Iniciar uma nova campanha de sustentabilidade - Descrição: Como gestor(a) do programa, quero iniciar uma nova campanha de sustentabilidade, para que possamos promover práticas sustentáveis na empresa. Critérios de Aceitação 1. Deve ser possível criar uma nova campanha de sustentabilidade. 2. Deve haver campos no qual serão colocados os dados sobre a campanha como: nome, descrição, metas de arrecadação e período. 3. Após o preenchimento dos dados, deve ser possível cadastrar a campanha. 4. Após o cadastro da campanha ser concluído, ela deve se tornar visível para os usuários do sistema. 5. O progresso da campanha poderá ser acompanhado. 6. Deve ser possível gerar relatórios sobre o desempenho da campanha. Fora do Escopo Acessando dados sobre a campanha - Descrição: Como gestor(a) do programa, quero acessar os dados sobre a campanha, para que eu me mantenha informado(a) do progresso da campanha. Critérios de Aceitação 1. Será possível visualizar uma lista de todas as campanhas. 2. Para cada campanha, devem ser exibidas informações, como: nome, descrição, data de início e de término, metas de arrecadação, a quantidade de doações feitas por cada colaborador. 3. Deve haver uma opção de filtrar, onde o usuário poderá escolher filtrar por: nome, status( em andamento ou concluída) ou período. 4. Ao selecionar uma campanha, deve ser possível visualizar relatórios sobre o desempenho da campanha. 5. Os relatórios serão visualizados em um painel centralizado. 6. Deve haver uma opção “Exportar”, onde os gráficos serão exportados em formato de planilhas. Fora do escopo 1. Permitir que os usuários compartilhem dados sobre as campanhas com outras pessoas nas redes sociais. Exportando informações sobre as campanhas - Descrição: Como gestor(a) do programa, quero exportar as informações sobre a campanha em formato de planilhas, para que eu possa fazer a análise dos dados e apresentar os dados para outras pessoas. Critérios de Aceitação 1. Ao clicar na opção “Exportar”, será possível exportar todas as informações sobre as campanhas para uma planilha. 2. As informações importadas devem incluir: nome, data de início e data de término, descrição, doações arrecadadas. 3. A planilha deve ser exportada de forma clara e organizada. Fora do Escopo 1. Permitir que os usuários formate a planilha de acordo com suas necessidades( ex: mudar a fonte, aplicar cores, etc). Acompanhando dados sobre as campanhas - Descrição: Como gestor(a) do programa, quero acompanhar os dados de campanhas em aberto ou anteriores, para que eu possa identificar tendências, avaliar o impacto das campanhas e possa tomar decisões mais conscientes. Critérios de Aceitação 1. Deve ser possível visualizar um painel com as principais métricas de desempenho para todas as campanhas de sustentabilidade em aberto e anteriores. 2. As métricas de desempenho devem incluir: nome, descrição, progresso, doações arrecadadas, etc. 3. Deve ser possível visualizar relatórios detalhados sobre o desempenho de cada campanha. Fora do Escopo 1. Comparar o desempenho das campanhas da organização com o desempenho de outras campanhas. Inserindo informações para incentivar os colaboradores - Descrição: Como gestor(a) do programa, quero inserir informações e artigos, para que eu possa motivar a participação dos colaboradores nas campanhas. Critérios de Aceitação 1. O sistema deve fornecer um portal de gestão de conteúdo para o programa de sustentabilidade. 2. O portal deve permitir que o gestor insira diferentes tipos de conteúdo sobre a campanha, como textos, imagens e arquivos PDF. 3. O sistema deve gerar automaticamente um feed de notícias com o conteúdo publicado. 4. Para a informação ou artigo ser incluído ele deve possuir: título, imagem(caso necessário), texto informativo. 5. Os colaboradores deverão receber notificações quando as informações forem inseridas. Fora do Escopo 1. Análise preditiva para identificar colaboradores com maior propensão a participar das campanhas. Recebendo doações de colaboradores - Descrição: Como gestor(a) do programa, quero receber doações de colaboradores e armazenar as informações em um dashboard centralizado, para que eu possa gerenciar as doações de forma eficiente. Critérios de Aceitação 1. O sistema deve permitir o registro de doações em diversas formas (Pix, produtos recicláveis, etc.). 2. O sistema deve armazenar as informações de cada doação, incluindo valor, data, tipo de doação e doador. 3. O sistema deve fornecer um dashboard que exiba uma visão geral das doações recebidas, incluindo valor total arrecadado, tipo de doação mais frequente e doadores mais generosos. 4. O sistema deve permitir que eu filtre e visualize as doações por diferentes critérios (data, tipo de doação, doador, etc.). 5. O sistema deve ser seguro e confiável, protegendo as informações dos doadores. Fora do Escopo 1. Análise preditiva para identificar colaboradores com maior propensão a participar das campanhas. Aderindo a uma nova campanha - Descrição: Como colaborador da empresa, quero aderir à nova campanha de forma fácil e prática, para que eu possa contribuir para um futuro mais sustentável. Critérios de Aceitação 1. O sistema deve permitir que os colaboradores se cadastrem na campanha de sustentabilidade de forma fácil e rápida. 2. O sistema deve apresentar as diferentes ações da campanha de forma clara e objetiva, com informações sobre como participar de cada uma delas. 3. O sistema deve permitir que os colaboradores se inscrevam nas ações da campanha que desejam participar. 4. O sistema deve enviar notificações e lembretes aos colaboradores sobre as ações da campanha em que se inscreveram. 5. O sistema deve permitir que os colaboradores acompanhem seu progresso individual e da empresa nas ações da campanha. 6. O sistema deve ser seguro e confiável, protegendo os dados dos colaboradores. Fora do Escopo 1. Geração de relatórios personalizados para cada colaborador, mostrando seu progresso individual e impacto nas ações da campanha. Contribuindo para campanha fazendo doações - Descrição: Como colaborador engajado, quero contribuir para a campanha de forma fácil e prática, para que eu possa fazer a diferença. Critérios de Aceitação 1. O sistema deve permitir que os colaboradores realizem doações para a campanha de forma simples e rápida. 2. O sistema deve oferecer diversas formas de doação, como Pix, produtos recicláveis, [outras formas de doação de acordo com a campanha]. 3. O sistema deve processar as doações com segurança e confiabilidade. 4. O sistema deve fornecer aos colaboradores um comprovante de doação. 5. O sistema deve permitir que os colaboradores acompanhem o histórico de suas doações. 6. O sistema deve ser seguro e confiável, protegendo os dados dos colaboradores. Fora do Escopo 1. Permita que os colaboradores configurem doações recorrentes, debitadas automaticamente de suas contas em intervalos regulares. Visualizando o Dashboard - Descrição: Como colaborador engajado, quero visualizar um dashboard com minhas contribuiçẽs para as campanhas da empresa, para que eu possa acompanhar meuprogresso individual e me sentir motivado a continuar contribuindo. Critérios de Aceitação 1. O sistema deve fornecer um dashboard acessível aos colaboradores. 2. O dashboard deve apresentar um resumo das contribuições do colaborador para as campanhas de sustentabilidade. 3. O resumo deve incluir informações sobre as diferentes formas de contribuição, como doações, economia de energia, reciclagem e participação em workshops. 4. As informações devem ser organizadas por período (mensal, semanal, geral) para facilitar a visualização do progresso. 5. O dashboard deve apresentar gráficos e indicadores visuais para tornar a visualização das informações mais intuitiva e atrativa. 6. O sistema deve permitir que o colaborador filtre as informações por campanha, período e tipo de contribuição. 7. O sistema deve ser seguro e confiável, protegendo os dados dos colaboradores. Fora do Escopo 1. Permitir que os colaboradores personalizem o dashboard, escolhendo quais informações deseja visualizar e como desejam que elas sejam apresentadas. Acessando dados sobre as campanhas - Descrição: Como colaborador engajado, quero acessar facilmente informações sobre as campanhas, para que eu possa me manter atualizado sobre as iniciativas em andamento e saber como posso contribuir. Critérios de Aceitação 1. O sistema deve fornecer um painel centralizado com gráficos sobre as campanhas de sustentabilidade da empresa. 2. Critérios de Aceitação 1. O sistema deve fornecer uma funcionalidade integrada para compartilhar informações sobre as campanhas de sustentabilidade nas redes sociais. 2. O colaborador deve escolher quais redes sociais deseja usar para compartilhar as informações, como Facebook, Instagram, Twitter e LinkedIn. 3. O sistema deve gerar um link único para cada compartilhamento, permitindo que as pessoas que clicarem no link sejam direcionadas para uma página de destino com mais informações sobre a campanha. 4. O sistema deve ser seguro e confiável, protegendo os dados dos colaboradores. Fora do Escopo 1. O sistema deve permitir que o colaborador personalize a mensagem de compartilhamento, incluindo texto, imagens e vídeos. STACK DE DESENVOLVIMENTO Inicialmente utilizaremos HTML, CSS e JavaScript.
